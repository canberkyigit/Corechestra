## Commands

```bash
npm start          # dev server (http://localhost:3000)
npm run build      # production build
npm test           # run tests (react-scripts test, interactive watch mode)
npm test -- --watchAll=false   # run tests once (CI mode)
```

Firebase is configured via environment variables — a `.env` file with `REACT_APP_FIREBASE_*` keys is required to run locally.

## Architecture Overview

Corechestra is a Jira/Confluence-like project management app built with React 19, Tailwind CSS, and Firebase.

### Auth & Provider Tree

```
AuthProvider (Firebase Auth state, role from Firestore `users` collection)
  └─ AuthGate
       └─ AppProvider (all project data, hydrated from Firestore)
            └─ ToastProvider
                 └─ HRProvider (HR-specific data from Firestore `hrData`)
                      └─ AppInner (routing + global modals)
```

- `AuthContext` exposes `user`, `role` (`admin` | `member` | `viewer`), `isAdmin`, `login`, `logout`, `updateProfile`.
- `AppContext` (accessed via `useApp()`) is the single source of truth for everything except auth and HR. New features should add state here.
- `HRContext` is self-contained; it reads/writes Firestore `hrData/{uid}` and `hrData/__shared__`.

### Firestore Data Model

All app state lives in the `appData` Firestore collection, split across ~10 domain documents to reduce write contention:

| Domain | Fields |
|---|---|
| `config` | currentUser, currentProjectId, darkMode, sidebar state, board filters |
| `entities` | projects, teams, users, epics, labels |
| `tasks` | activeTasks, perProjectBacklog |
| `sprints` | perProjectSprint, projectColumns, burndown snapshots, planned/completed sprints |
| `activity` | globalActivityLog, notifications |
| `workspace` | perProjectRetrospective, perProjectPokerHistory, perProjectNotes |
| `docs` | spaces, docPages |
| `releases` | releases |
| `testing` | testSuites, testCases, testRuns |
| `archive` | archivedTasks, archivedProjects, archivedEpics |

**Write path**: Always go through `saveDomain(domain, data)` in `src/services/storage.js`. Writes are debounced 1500ms. **Never write to Firestore directly from components.**

**Real-time sync**: `subscribeToAll(onUpdate)` listens for remote changes and ignores own writes (via `_updatedAt` timestamp comparison).

User auth profiles and roles live in the separate `users/{uid}` Firestore collection (managed by `AuthContext`).

### Per-Project State Pattern

Most mutable data is stored as maps keyed by `currentProjectId`:

```js
perProjectSprint[currentProjectId]      // active sprint
perProjectBacklog[currentProjectId]     // backlog sections
projectColumns[currentProjectId]        // kanban columns
perProjectBoardSettings[currentProjectId]
```

AppContext exposes scoped getters and setters (e.g., `sprint`, `setSprint`, `backlogSections`) that automatically scope to `currentProjectId`.

### Routing

All routes are defined in `src/App.js`. The `PATH_TO_PAGE` map drives the `activePage` prop passed to `Layout`. Admin route is guarded by `isAdmin`. All pages are wrapped in a `PageTransition` (framer-motion fade/slide).

### Dark Mode

Tailwind `darkMode: 'class'`. The `dark` class is toggled on `document.documentElement`. Use these hardcoded dark bg/border colors consistently:

- Background layers: `bg-[#080b14]` (deepest), `bg-[#141720]`, `bg-[#1a1f2e]`, `bg-[#1c2030]`, `bg-[#232838]`
- Borders: `border-[#2a3044]`, `border-[#252b3b]`

All new UI must support both light and dark modes using Tailwind's `dark:` prefix.

### Task Model

- Task IDs use the format `CY-XXXXX` (generated by `generateId()` in `src/utils/helpers.js`). Use `taskKey(id)` for display to avoid `CY-CY-` double prefixes.
- Task types: `task`, `bug`, `feature`, `defect`, `userstory`, `investigation`, `epic`, `test`, `testset`, `testexecution`, `precondition`
- Statuses: `todo`, `inprogress`, `review`, `awaiting`, `blocked`, `done`
- Priorities: `critical`, `high`, `medium`, `low`
- Viewers (`role === "viewer"`) cannot create or modify tasks.

### Key Files

- `src/context/AppContext.js` — monolithic app state + Firestore sync
- `src/context/AppSeeds.js` — `SEED_*` and `DEFAULT_*` constants (default columns, sprint defaults, seed data)
- `src/services/storage.js` — Firestore read/write/subscribe API
- `src/services/firebase.js` — Firebase app init (config from env vars)
- `src/components/Layout.js` — sidebar nav, notification panel (`NOTIF_META` lookup), search
- `src/constants/taskOptions.js` — `TYPE_OPTIONS`, `STATUS_OPTIONS`, `PRIORITY_OPTIONS`
- `src/utils/helpers.js` — `generateId()`, `taskKey()`, `generateSubtasks()`, `getPriorityColor()`

### Notifications

Notifications are event-driven from AppContext. When adding a new notification type, add the type + icon + color to the `NOTIF_META` map in `src/components/Layout.js`.

### Docs (Confluence-like)

`src/pages/DocsPage.js` (~800 lines) implements a 2-panel layout with hierarchical page tree, MDEditor (`@uiw/react-md-editor`), TipTap rich editor (`@tiptap/react` + extensions for mentions and images), drag-and-drop reordering (`@hello-pangea/dnd`), and per-page comments.
